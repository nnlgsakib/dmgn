# Plan 06-03: Backup/Restore + Peer Reputation

<objective>
Implement encrypted backup export/restore for full node recovery and peer reputation scoring for network quality management.
</objective>

## Tasks

<task id="06-03-01" title="Create backup package — export">
<read_first>
- pkg/backup/backup.go (new file)
- pkg/storage/storage.go (Store.db, Backup method pattern)
- pkg/vectorindex/index.go (VectorIndex.Export)
</read_first>
<action>
Create `pkg/backup/backup.go` with:

1. `BackupConfig` struct: `OutputPath string`, `Store *storage.Store`, `VecIndex *vectorindex.VectorIndex`, `Config *config.Config`, `NodeID string`

2. `Export(cfg BackupConfig) error`:
   - Create temp dir
   - Write `manifest.json`: `{version, node_id, timestamp, dmgn_version}`
   - Stream BadgerDB backup to `badger.backup` file via `db.Backup(w, 0)`
   - Export vector index to `vector-index.bin` via `vecIndex.ExportRaw(w)`
   - Write `config.json` (sanitized — no keys/secrets)
   - Create tar.gz from temp dir files → `OutputPath`
   - Clean up temp dir
   - Return nil on success

3. `ExportRaw(w io.Writer) error` method on VectorIndex — write raw binary index data (already encrypted internally). If this doesn't exist, add it.
</action>

<task id="06-03-02" title="Create backup package — restore">
<read_first>
- pkg/backup/backup.go (just created)
- pkg/storage/storage.go (Store constructor, db.Load)
</read_first>
<action>
Add to `pkg/backup/backup.go`:

1. `RestoreConfig` struct: `InputPath string`, `DataDir string`

2. `Restore(cfg RestoreConfig) (*RestoreResult, error)`:
   - Extract tar.gz to temp dir
   - Read and validate `manifest.json` (check version compatibility)
   - Restore BadgerDB: open new DB at `DataDir/storage`, call `db.Load(r, 256)`
   - Restore vector index: copy `vector-index.bin` to `DataDir/vector-index.enc`
   - Restore config: merge restored config with current (don't overwrite identity)
   - Clean up temp dir
   - Return `RestoreResult{NodeID, MemoryCount, Timestamp}`

3. `RestoreResult` struct: `NodeID string`, `MemoryCount int64`, `Timestamp time.Time`, `Version string`
</action>

<action>
Create `internal/cli/backup.go` with two commands:

1. `BackupCmd() *cobra.Command`:
   - Usage: `dmgn backup [output-file]`
   - Flags: `--data-dir`
   - Default output: `dmgn-backup-{timestamp}.dmgn-backup`
   - Load config, identity, open storage and vector index (read-only)
   - Call `backup.Export(cfg)`

   - Flags: `--data-dir`, `--force` (overwrite existing)
   - Check if data dir has existing data, warn if so (require --force)
   - Call `backup.Restore(cfg)`
   - Print: `✓ Restored: {node_id}, {memory_count} memories from {timestamp}`

Register both in `cmd/dmgn/main.go`:

<task id="06-03-04" title="Create peer reputation module">
<read_first>
- pkg/network/reputation.go (new file)
- pkg/network/peers.go (PeerConnectionInfo, ConnectedPeers)
- pkg/storage/storage.go (BadgerDB access via DB())
</read_first>
<action>
Create `pkg/network/reputation.go` with:

1. `PeerReputation` struct:
   - `PeerID string`
   - `UptimeRatio float64` (0-1)
   - `AvgLatencyMs float64`
   - `SyncSuccessRate float64` (0-1)
   - `DataAvailability float64` (0-1)
   - `Score float64` (computed)
   - `LastSeen time.Time`
   - `InteractionCount int64`

2. `ReputationManager` struct: holds `*badger.DB`, `mu sync.RWMutex`, `cache map[string]*PeerReputation`

3. `NewReputationManager(db *badger.DB) *ReputationManager`

4. `RecordInteraction(peerID string, latencyMs float64, success bool)`:
   - Update rolling averages for the peer
   - Recompute score: `0.3*uptime + 0.3*latencyScore + 0.2*syncRate + 0.2*availability`
   - Latency score: `max(0, 1.0 - avgLatency/5000.0)`
   - Save to BadgerDB key `rep:{peer_id}`

5. `GetScore(peerID string) float64` — return cached score or load from DB

6. `GetTopPeers(n int) []PeerReputation` — return n peers sorted by score descending

7. `DecayScores(factor float64)` — move all scores toward 0.5 by factor (called periodically)

8. `Save(peerID string) error` / `Load(peerID string) (*PeerReputation, error)` — BadgerDB persistence with JSON marshaling, key prefix `rep:`
</action>
<acceptance_criteria>
- `pkg/network/reputation.go` exists with `ReputationManager` struct

- BadgerDB key prefix is `rep:`
- `go build ./pkg/network/...` exits 0
</acceptance_criteria>
</task>
