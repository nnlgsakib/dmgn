Create new file `internal/daemon/keys.go`:

1. Define `DerivedKeys` struct with fields:
   - `MasterKey []byte` — for memory encryption (AES-GCM)
   - `LibP2PKey crypto.PrivKey` — pre-derived libp2p ed25519 key
   - `VectorIndexKey []byte` — for vector index encryption
2. Add `DeriveAll(id *identity.Identity) (*DerivedKeys, error)` function that:
   - Calls `id.DeriveKey("memory-encryption", 32)` for MasterKey
   - Calls `network.DeriveLibp2pKey(id)` for LibP2PKey
   - Calls `id.DeriveKey("vector-index", 32)` for VectorIndexKey
3. Add `Encode() string` method — base64 encodes all keys for env var transport
4. Add `Decode(encoded string) (*DerivedKeys, error)` — inverse of Encode

The Encode/Decode are needed because `dmgn start` (foreground) derives keys, then passes
them to the background daemon child process via environment variable `DMGN_DERIVED_KEYS`.

    - Encode/Decode round-trip produces identical keys
  </acceptance_criteria>

3. `func (d *Daemon) Start(ctx context.Context) error` — port the ENTIRE wiring from
   start.go lines 56-261:
   - Create crypto engine from keys.MasterKey
   - Open storage
   - Create libp2p host from keys.LibP2PKey (using cfg.ListenAddr, cfg.BootstrapPeers, etc.)
   - Start host (DHT + mDNS)
   - Register shard handlers, create distributor, start auditor
   - Create vector index, load it
   - Create query engine, remote orchestrator, register handler
   - Set up version vector, gossip manager, delta sync manager
   - Create API server, wire everything, start it
   - Create MCP server with all dependencies
   - Start TCP MCP IPC listener on 127.0.0.1:{MCPIPCPort} (auto-assign if 0)
   - Write assigned port to cfg.PortFile()
   - Accept MCP connections in goroutine

4. `func (d *Daemon) Stop() error` — graceful shutdown:
   - Close MCP listener
   - Stop API server (5s timeout)
   - Stop gossip, delta sync
   - Save vector index
   - Close storage
   - Stop libp2p host
   - Remove port file

5. `func (d *Daemon) handleMCPConnection(conn net.Conn)` — for each MCP IPC
   connection, create a context and run mcpServer.RunOnConnection(ctx, conn).
   This requires Plan 03 to add RunOnConnection to pkg/mcp/server.go, so for now
   use a TODO placeholder that we'll wire in Plan 03.

CRITICAL: This is a MOVE of logic from start.go, not new logic. The subsystem wiring
must match exactly what start.go currently does (lines 56-261). Log to file via
slog.Logger writing to lumberjack (already in go.mod) at cfg.LogDir()/daemon.log.
