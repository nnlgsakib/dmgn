  <name>Complete daemon MCP TCP listener</name>
  <files>internal/daemon/daemon.go</files>
  <action>
Replace the TODO placeholder from Plan 01 with actual MCP IPC handling:

1. In daemon.Start(), after creating MCP server:
   - Listen on 127.0.0.1:{MCPIPCPort} (0 = auto-assign)
   - Store assigned port to cfg.PortFile()
   - Start accept loop in goroutine

2. `func (d *Daemon) acceptMCPConnections()`:
   ```go
   for {
       conn, err := d.mcpListener.Accept()
       if err != nil {
           if d.ctx.Err() != nil { return } // shutting down
           d.logger.Error("MCP accept error", "err", err)
           continue
       }
       go d.handleMCPConnection(conn)
   }
   ```

3. `func (d *Daemon) handleMCPConnection(conn net.Conn)`:
   - Create per-connection context (canceled when daemon stops or conn closes)
   - Call d.mcpServer.RunOnConnection(ctx, conn)
   - Log connection open/close

4. In daemon.Stop(), close mcpListener to break accept loop
  </action>

    - Port file is written after listener starts
    - Graceful shutdown closes listener and active connections
  </acceptance_criteria>
  <done>Daemon serves MCP over TCP IPC</done>

<task type="auto">
  <name>Create dmgn mcp stdio proxy command</name>
  <files>internal/cli/mcp_proxy.go</files>
  <action>
Create new file `internal/cli/mcp_proxy.go`:

1. `func MCPCmd() *cobra.Command` returning a cobra command for `dmgn mcp`:
   ```
   Use:   "mcp"
   Short: "Connect to DMGN daemon via MCP (stdio)"
   Long:  "Bridges stdin/stdout to the running DMGN daemon's MCP server.
           Configure AI tools to run: dmgn mcp"
   ```

2. RunE implementation:
   a. Load config to get DataDir
   b. Read port from cfg.PortFile()
   c. If port file doesn't exist or daemon not running:
      - Print to STDERR: "DMGN daemon is not running. Start it with: dmgn start"
      - Exit with code 1
   d. Dial TCP to 127.0.0.1:{port}
   e. Bridge bidirectionally:
      - goroutine: io.Copy(conn, os.Stdin)
      - main: io.Copy(os.Stdout, conn)
   f. When either copy returns, close both sides

3. Flags: `--data-dir` (same as other commands)

This is the command AI tools will spawn. It MUST:
- Never write anything to stdout except MCP JSON-RPC (stdout is the MCP channel)
- All errors/logs go to stderr only
- Exit cleanly when stdin is closed (AI tool disconnects)
  </action>

    - stderr gets error messages (never stdout)
    - Clean exit on stdin close
    - Clear error if daemon not running
  </acceptance_criteria>
  <done>MCP stdio proxy ready for AI tool configuration</done>
