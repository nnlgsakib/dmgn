<objective>
Restructure the CLI to use the new daemon architecture. `dmgn start` becomes a background
daemon launcher (prompts passphrase, derives keys, spawns detached child, exits). `dmgn stop`
gracefully terminates the daemon. Old `mcp-serve` and `serve` commands are removed. The
`dmgn mcp` proxy (from Plan 03) is registered. A `--foreground` flag is available for debugging.
</objective>

<task type="auto">
  <name>Rewrite dmgn start as background launcher</name>
  <files>internal/cli/start.go</files>
  <action>
Completely rewrite `internal/cli/start.go`:

1. `func StartCmd() *cobra.Command` with flags:
   - `--data-dir` (string, existing)
   - `--foreground` (bool, new) — run daemon in foreground for debugging
   - `--daemon-mode` (bool, HIDDEN) — internal flag, means "I am the child daemon process"

2. RunE logic — TWO paths:

   **Path A: --daemon-mode (child daemon process)**
   - Read DMGN_DERIVED_KEYS env var
   - Decode into DerivedKeys
   - Load config
   - Create Daemon: daemon.New(cfg, keys)
   - Write PID file
   - Start daemon: d.Start(ctx)
   - Block on SIGTERM/SIGINT
   - On signal: d.Stop(), remove PID file, exit

   **Path B: Normal start (parent launcher)**
   - Load config
   - Check if daemon already running via CheckDaemonRunning(cfg.PIDFile())
     - If running: print "Daemon already running (PID: X)" and exit
   - Check identity exists
   - Prompt passphrase
   - Load identity, derive all keys: daemon.DeriveAll(id)
   - If --foreground: run daemon directly (same as --daemon-mode but in current process)
   - Else: spawn background daemon:
     - Encode keys to env var
     - Call SpawnDaemon(os.Args[0], ["start", "--daemon-mode", "--data-dir", dataDir], env)
     - Call WaitForHealthy(cfg.PIDFile(), 10s)
     - Print success message:
       ```
       DMGN daemon started (PID: XXXX)
       MCP IPC port: YYYY
       To stop: dmgn stop
       ```
     - Exit parent

3. Remove ALL old start.go code — the subsystem wiring now lives in internal/daemon/daemon.go
  </action>
