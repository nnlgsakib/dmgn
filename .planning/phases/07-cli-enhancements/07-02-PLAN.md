Create new file `internal/daemon/pidfile.go`:

1. `func WritePID(pidPath string) error`
   - Writes current process PID to file
   - Creates parent dirs if needed
   - File contains just the PID number as ASCII

2. `func ReadPID(pidPath string) (int, error)`
   - Reads PID from file
   - Returns 0 + error if file doesn't exist or is malformed

3. `func CheckDaemonRunning(pidPath string) (int, bool)`
   - Reads PID file
   - Checks if process with that PID is still alive (calls isProcessAlive)
   - Returns (pid, true) if daemon is running
   - Returns (0, false) if no PID file or stale PID

4. `func RemovePID(pidPath string) error`
   - Removes PID file, ignoring not-exist errors

5. `func isProcessAlive(pid int) bool`
   - Platform-specific: uses build tags
   - Unix: syscall.Kill(pid, 0) == nil
   - Windows: OpenProcess with PROCESS_QUERY_LIMITED_INFORMATION

<task type="auto">
  <name>Create process spawn/kill utilities</name>
  <files>internal/daemon/process.go, internal/daemon/process_unix.go, internal/daemon/process_windows.go</files>
  <action>
Create cross-platform process management using build tags:

**internal/daemon/process.go** (shared interface):
1. `func SpawnDaemon(execPath string, args []string, env []string) (int, error)`
   - Spawns a detached child process using os/exec
   - Sets platform-specific SysProcAttr for detach
   - Returns child PID
   - Does NOT wait for child (fire and forget)

2. `func StopDaemon(pidPath string, timeout time.Duration) error`
   - Reads PID from file
   - Sends graceful signal (SIGTERM on Unix, TerminateProcess on Windows)
   - Polls for process exit up to timeout
   - If still alive after timeout, force kills
   - Removes PID file and port file

3. `func WaitForHealthy(pidPath string, timeout time.Duration) error`
   - Polls until PID file exists and process is alive
   - Used by `dmgn start` to confirm child daemon started successfully
   - Returns error if timeout exceeded

**internal/daemon/process_unix.go** (build tag: !windows):
- `func detachAttrs() *syscall.SysProcAttr` → returns `{Setsid: true}`
- `func signalProcess(pid int) error` → `syscall.Kill(pid, syscall.SIGTERM)`
- `func forceKillProcess(pid int) error` → `syscall.Kill(pid, syscall.SIGKILL)`

**internal/daemon/process_windows.go** (build tag: windows):
- `func detachAttrs() *syscall.SysProcAttr` → returns `{HideWindow: true, CreationFlags: 0x00000008}`
- `func signalProcess(pid int) error` → uses taskkill or TerminateProcess via syscall
- `func forceKillProcess(pid int) error` → same with force flag
