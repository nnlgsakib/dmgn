<objective>
Fix the crypto engine's decrypt length calculation bug, add length-prefixed encrypted key framing, replace the weak SHA256-based key derivation with HKDF-SHA256 (per Trail of Bits best practices and RFC 5869), add configurable memory retention to the storage layer, and add comprehensive crypto tests.
</objective>

<research_findings>
See RESEARCH.md sections 1 (HKDF Required), 2 (Framing Bug), 6 (Merkle = per-memory hash).
- Current Identity.DeriveKey uses raw SHA256 concatenation — not cryptographically sound
- HKDF-SHA256 via golang.org/x/crypto/hkdf is the correct replacement
- Envelope encryption framing has encryptedKeyLen=28 bug (should be ~60 bytes, or use length prefix)
- Phase 1 data will be incompatible — accepted as dev-only break
</research_findings>

<must_haves>
- Crypto engine correctly encrypts and decrypts payloads round-trip
- Each memory gets a unique per-memory AES-GCM-256 key
- Per-memory keys are encrypted with master key derived from identity via HKDF
- Key derivation uses HKDF-SHA256 with unique info parameter per purpose
- Configurable retention limit (N most recent memories kept)
- Memory.ID = SHA256(encrypted_payload) verified on load (CRPT-04)
</must_haves>

<read_first>
- pkg/identity/identity.go (current DeriveKey method uses sha256(seed + purpose) with iterative hashing)
- RESEARCH.md section 1 (HKDF Required)
</read_first>

<action>
Rewrite `Identity.DeriveKey` in `pkg/identity/identity.go`:

1. Add import: `"golang.org/x/crypto/hkdf"` and `"io"`
2. Replace the method:

```go
func (i *Identity) DeriveKey(purpose string, keyLen int) ([]byte, error) {
    ikm := i.PrivateKey.Seed()           // 32-byte ed25519 seed = high-entropy IKM
    info := []byte(purpose)               // unique per subkey (domain separation)
    reader := hkdf.New(sha256.New, ikm, nil, info) // nil salt per RFC 5869 for high-entropy IKM
    key := make([]byte, keyLen)
    if _, err := io.ReadFull(reader, key); err != nil {
        return nil, fmt.Errorf("HKDF key derivation failed: %w", err)
    }
    return key, nil
}
```

3. Update all callers (signature change: old `DeriveKey(purpose string, index uint32) []byte` → new `DeriveKey(purpose string, keyLen int) ([]byte, error)`):

   - `internal/cli/query.go`: `masterKey, err := id.DeriveKey("memory-encryption", 32)`
   - Any other caller found via grep

4. Update `pkg/identity/identity_test.go`:
   - Test determinism: same identity + same purpose → same key
   - Test separation: same identity + different purpose → different keys
   - Test key length: request 16, 32, 64 bytes → correct lengths returned
</action>

### Task 1: Fix crypto engine decrypt framing

<read_first>
- internal/crypto/crypto.go (current implementation with hardcoded encryptedKeyLen=28)
</read_first>

<action>
Fix `internal/crypto/crypto.go` Decrypt method. The current `encryptedKeyLen := 28` is wrong — AES-GCM with 12-byte nonce encrypting a 32-byte key produces 12 (nonce) + 32 (ciphertext) + 16 (tag) = 60 bytes.

Replace the hardcoded length with a length-prefixed framing scheme:

In `Encrypt()`:
1. After `encryptedKey` is produced, prepend a 2-byte big-endian length: `binary.BigEndian.PutUint16(buf, uint16(len(encryptedKey)))`
2. Result format: `[2-byte key len][encrypted key][encrypted payload]`

In `Decrypt()`:
1. Read first 2 bytes as `keyLen := binary.BigEndian.Uint16(ciphertext[:2])`
2. Extract `encryptedKey = ciphertext[2:2+keyLen]`
3. Extract `encryptedPayload = ciphertext[2+keyLen:]`
4. Validate: `if len(ciphertext) < int(keyLen)+2 { return error }`

This makes the format self-describing and immune to nonce size changes.
</action>

<acceptance_criteria>
- `crypto.go` contains `binary.BigEndian.PutUint16` in Encrypt method
- `crypto.go` contains `binary.BigEndian.Uint16` in Decrypt method
- `crypto.go` does NOT contain `encryptedKeyLen := 28`
- Round-trip test: `Decrypt(Encrypt(plaintext)) == plaintext` for payloads of 0, 1, 1000, and 65536 bytes
</acceptance_criteria>

### Task 3: Add configurable retention to storage

<read_first>
- pkg/storage/storage.go (current Store struct and Options)
- internal/config/config.go (MaxRecentMemories field)
</read_first>

<action>
1. Add `MaxRetention int` field to `storage.Options` struct (default 0 = unlimited)
2. Add `maxRetention int` field to `Store` struct, set from Options in `New()`
3. Create `pkg/storage/retention.go` with:

```go
func (s *Store) EnforceRetention() (int, error)
```

This method:
- If `s.maxRetention <= 0`, return 0, nil (unlimited)
- Count total memories using PrefixMemory iteration
- If count <= maxRetention, return 0, nil
- Otherwise, iterate time index (oldest first), collect IDs to delete
- Delete excess memories (count - maxRetention oldest)
- Return number deleted, nil

4. Call `EnforceRetention()` at the end of `SaveMemory()` when maxRetention > 0
5. Update `internal/config/config.go`: ensure `MaxRecentMemories` maps to storage retention
</action>
