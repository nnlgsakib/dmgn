### Task 1: Enhance CLI query with scoring and output format

<read_first>
- internal/cli/query.go (current searchMemories function with simple substring match)
- pkg/memory/memory.go (Memory struct, PlaintextMemory, Decrypt method)
</read_first>

<action>
Update internal/cli/query.go:

1. Replace searchMemories with a scored search:

Define scoredResult struct with fields: memory *memory.Memory, plain *memory.PlaintextMemory, score float64.

Define func searchMemoriesScored(store, query, limit, decryptFn) returning []scoredResult.

Scoring algorithm:
- Exact match in content: score 1.0
- Case-insensitive contains: score 0.8
- Query words present (more than 50% of words match): score 0.5
- Partial word match: score 0.3
- Sort results by score descending

2. Update output format to include score:
   Format each result as: 1. [0.80] content preview...
   Below: ID: abc123... | Type: text | Links: 0 | timestamp formatted

3. Add --format flag: text (default) or json for machine-readable output.
   JSON format outputs {"results":[{"id":"...","content":"...","score":0.8}]} to stdout.
</action>

<acceptance_criteria>
- query.go contains func searchMemoriesScored
- query.go contains type scoredResult struct
- Query output includes score in brackets like [0.80]
- --format json flag exists and produces valid JSON to stdout
- Results are sorted by score descending
</acceptance_criteria>
