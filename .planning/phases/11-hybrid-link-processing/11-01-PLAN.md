  <action>
Add a new function `autoLinkNewMemory` that handles auto-linking logic. Insert it after the existing `handleAddMemory` function (~line 304).

The function should:

1. Skip if `cfg.EnableAutoLink` is false
2. Skip if new memory has no embedding (use time-clustering only in that case)
3. Use `vecIndex.Search()` to find top-k memories by cosine similarity
4. For each similar memory above threshold:
   - Create edge with edge_type = "auto" and weight = similarity score
   - Store via `store.AddEdge()`
   - Broadcast via `edgeBroadcaster()` (check for nil)
5. Also add time-clustering edges for memories within time window

Add an explicit `edgeBroadcaster` field or method on MCPServer if not present. Check existing code for how edge broadcaster is passed (line 406-407 shows `s.edgeBroadcaster`).

**CRITICAL - Implementation per D-XX:**
- Per D-01: Trigger happens immediately on add_memory
- Per D-03: Hybrid algorithm checks BOTH similarity AND time clustering
- Per D-04: Similarity threshold default 0.7, configurable via config
- Per D-06: Edge weight = similarity score (0.0-1.0 preserves confidence)
- Per D-08: Auto-edges broadcast to network via gossip (same as manual)
- Per D-09: Edge type field differentiates "auto" vs "manual"
