**Phase:** 05-query-sync
**Wave:** 2 (requires Wave 1: vectorindex, vclock, config)

## Objective

Build the local+remote query engine with hybrid scoring, integrate GossipSub for real-time memory propagation, and implement the delta sync protocol for reconnected peers. This wave wires the Wave 1 foundations into the network layer.

- **SYNC-02:** Propagate new memories to connected peers
- **SYNC-03:** Eventual consistency model with conflict resolution
- **SYNC-04:** Efficient delta sync for reconnected peers

### Task 1: Local Query Engine with Hybrid Scoring

**File:** `pkg/query/engine.go`

```go
package query

// QueryEngine handles local and remote memory search.
type QueryEngine struct {
    index     *vectorindex.VectorIndex
    store     *storage.Store
    decryptFn func([]byte) ([]byte, error)
    alpha     float64  // hybrid score weight (default 0.7)
}

// NewQueryEngine creates a query engine with a vector index and storage backend.
func NewQueryEngine(index *vectorindex.VectorIndex, store *storage.Store,
    decryptFn func([]byte) ([]byte, error), alpha float64) *QueryEngine

// QueryRequest holds parameters for a query.
type QueryRequest struct {
    Embedding []float32         // vector query (optional if TextQuery set)
    TextQuery string            // text query (optional if Embedding set)
    Limit     int               // max results (default 10)
    Filters   QueryFilters      // optional filters
}

type QueryFilters struct {
    Type   string // memory type filter (empty = all)
    After  int64  // timestamp lower bound (0 = no filter)
    Before int64  // timestamp upper bound (0 = no filter)
}

// QueryResult holds a single result.
type QueryResult struct {
    MemoryID   string  `json:"memory_id"`
    Score      float64 `json:"score"`
    Type       string  `json:"type"`
    Timestamp  int64   `json:"timestamp"`
    Snippet    string  `json:"snippet"`     // first 100 chars of decrypted content
    SourcePeer string  `json:"source_peer"` // empty for local results
}

// SearchLocal performs a local hybrid search (vector + text).
func (qe *QueryEngine) SearchLocal(req QueryRequest) ([]QueryResult, error)
```

Implementation details:
- If `req.Embedding` is set: get top-K*2 from HNSW, then compute hybrid scores
- If only `req.TextQuery` is set: fall back to text-only search (existing scoreMatch logic)
- Hybrid score: `α * vector_score + (1-α) * text_score`
- For each HNSW result, decrypt memory, compute text score, blend
- Apply filters (type, time range) before returning
- Generate snippet: first 100 chars of decrypted content
- Sort by final score descending, return top Limit

### Task 2: GossipSub Integration

**File:** `pkg/sync/gossip.go`

```go
package sync

import (
    pubsub "github.com/libp2p/go-libp2p-pubsub"
)

// GossipMessage is the envelope for gossip broadcasts.
type GossipMessage struct {
    Type         string `json:"type"`           // "new_memory"
    Memory       []byte `json:"memory"`         // JSON-encoded Memory struct
    SenderPeerID string `json:"sender_peer_id"`
    Timestamp    int64  `json:"timestamp"`
    Sequence     uint64 `json:"sequence"`       // sender's sequence number
}

// GossipManager handles pubsub memory propagation.
type GossipManager struct {
    ps          *pubsub.PubSub
    topic       *pubsub.Topic
    sub         *pubsub.Subscription
    topicName   string
    localPeerID string
    onReceive   func(msg *GossipMessage) // callback for received memories
    cancel      context.CancelFunc
    done        chan struct{}
}

// NewGossipManager creates a GossipSub manager for a libp2p host.
func NewGossipManager(ctx context.Context, host host.Host, topicName string,
    localPeerID string, onReceive func(msg *GossipMessage)) (*GossipManager, error)

// Publish broadcasts a new memory to the gossip network.
func (gm *GossipManager) Publish(ctx context.Context, mem *memory.Memory, seq uint64) error

// Start begins listening for incoming gossip messages in a goroutine.
func (gm *GossipManager) Start(ctx context.Context)

// Stop shuts down the gossip manager.
func (gm *GossipManager) Stop()
```

Implementation details:
- Create GossipSub: `pubsub.NewGossipSub(ctx, host)`
- Join topic: `ps.Join(topicName)`
- Subscribe: `topic.Subscribe()`
- Register message validator: validate JSON, reject own messages (avoid echo)
- `Publish()`: marshal `GossipMessage` envelope → `topic.Publish(ctx, bytes)`
- Message loop: `sub.Next(ctx)` → unmarshal → call `onReceive` callback
- The `onReceive` callback (set by caller in start.go) stores the memory locally, indexes the embedding, and updates the version vector

### Task 3: Delta Sync Protocol

**File:** `pkg/sync/delta.go`

```go
// DeltaSyncManager handles version-vector-based reconnection sync.
type DeltaSyncManager struct {
    host        host.Host
    vv          *VersionVector
    vvStore     *VClockStore
    store       *storage.Store
    localPeerID string
    syncInterval time.Duration
    cancel      context.CancelFunc
    done        chan struct{}
    onReceive   func(mem *memory.Memory) // callback for received memories
}

// SyncProtocol is the protocol ID for delta sync.
const SyncProtocol = protocol.ID("/dmgn/memory/sync/1.0.0")

// NewDeltaSyncManager creates a new delta sync manager.
func NewDeltaSyncManager(host host.Host, vv *VersionVector, vvStore *VClockStore,
    store *storage.Store, localPeerID string, syncInterval time.Duration,
    onReceive func(mem *memory.Memory)) *DeltaSyncManager

// RegisterHandler registers the /dmgn/memory/sync/1.0.0 stream handler.
func (dm *DeltaSyncManager) RegisterHandler()

// SyncWithPeer initiates delta sync with a specific peer.
func (dm *DeltaSyncManager) SyncWithPeer(ctx context.Context, peerID peer.ID) error

// Start begins periodic sync and registers the connected notifiee.
func (dm *DeltaSyncManager) Start(ctx context.Context)

// Stop shuts down the delta sync manager.
func (dm *DeltaSyncManager) Stop()
```

Sync protocol flow:
1. **Initiator** opens stream to `/dmgn/memory/sync/1.0.0`
2. **Initiator** sends own version vector (JSON frame via existing `writeFrame`)
3. **Responder** compares vectors, identifies missing ranges
4. **Responder** sends batch of missing memories (JSON array frame)
5. **Responder** sends own version vector
6. **Initiator** compares, sends what responder is missing
7. Both sides call `onReceive` for each received memory, update local VV

Triggers:
- On peer connected (via network.Notifiee): debounce 2s then `SyncWithPeer`
- Periodic ticker at `syncInterval` (default 60s): sync with all connected peers
- Manual via future CLI command (deferred)
