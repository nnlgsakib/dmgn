# Plan 03-02: CLI Commands — start, peers, status

<objective>
Wire the network package into CLI commands. Replace the `dmgn start` stub with a full libp2p host + optional API server. Add `dmgn peers` command. Update `dmgn status` with live peer info.
</objective>

<must_haves>
- `dmgn start` launches libp2p host + REST API (with --no-api flag)
- `dmgn start` prints peer ID and listen addresses on startup
- `dmgn start` gracefully shuts down on SIGINT/SIGTERM
- `dmgn peers` lists connected peers with IDs and addresses
- `dmgn status` shows live peer count and network status
- PeersCmd registered in main.go
</must_haves>

## Tasks

<task id="1">
<title>Replace dmgn start with Full Networking</title>

<read_first>
- internal/cli/start.go
- internal/cli/serve.go (API server pattern to reuse)
- pkg/network/host.go
- internal/config/config.go
</read_first>

<action>
Rewrite `internal/cli/start.go` to:

1. Load config and identity (prompt passphrase via `promptPassphraseOnce()`)
2. Derive libp2p key: `network.DeriveLibp2pKey(id)`
3. Create host config from config fields:
   ```go
   hostCfg := network.HostConfig{
       ListenAddrs:    []string{cfg.ListenAddr},
       BootstrapPeers: cfg.BootstrapPeers,
       MDNSService:    cfg.MDNSService,
       MaxPeersLow:    cfg.MaxPeersLow,
       MaxPeersHigh:   cfg.MaxPeersHigh,
       PrivateKey:     privKey,
   }
   ```
4. Create and start host: `network.NewHost(hostCfg)` then `host.Start()`
5. Print peer ID and listen addresses
6. Unless `--no-api` flag set, start API server (reuse `api.NewServer` from serve.go pattern):
   - Derive API key via HKDF
   - Open storage
   - Create and start API server on cfg.APIPort
   - Print API key
7. Wait for SIGINT/SIGTERM
8. Graceful shutdown: stop API server (if running), stop host, close storage

Add flags:
- `--no-api` (bool, default false) — skip REST API server
- `--data-dir` (string) — data directory override
</action>

- `internal/cli/start.go` contains `host.Start()`

- `internal/cli/start.go` contains `api.NewServer` (conditional on --no-api)

<task id="2">
<title>Create dmgn peers Command</title>

<read_first>
- pkg/network/peers.go
- internal/cli/status.go (pattern reference)
</read_first>

<action>
Create `internal/cli/peers.go` with `PeersCmd() *cobra.Command`:

The peers command needs access to a running host. Since the node runs as a foreground process, `dmgn peers` must query via the REST API.

Add a new API endpoint in this plan or use a simpler approach:

**Approach:** `dmgn peers` connects to the local API to get peer info.

1. Add `GET /peers` endpoint to `internal/api/handlers.go`:
   - Return JSON array of connected peers from the network host
   - Requires the server to have a reference to the network host

2. Create `internal/cli/peers.go`:
   ```go
   func PeersCmd() *cobra.Command {
       var apiAddr string
       cmd := &cobra.Command{
           Use:   "peers",
           Short: "List connected peers",
           RunE: func(cmd *cobra.Command, args []string) error {
               // HTTP GET to local API /peers endpoint
               // Parse and display peer list
           },
       }
       cmd.Flags().StringVar(&apiAddr, "api", "http://localhost:8080", "API server address")
       return cmd
   }
   ```

3. Output format:
   ```
   Connected Peers (N)
   ==================
   
   Peer ID                    Addresses
   -------                    ---------
   12D3KooW...abc             /ip4/192.168.1.5/tcp/4001
   12D3KooW...def             /ip4/10.0.0.2/tcp/4001
   ```

4. Register `PeersCmd()` in `cmd/dmgn/main.go`
</action>

<task id="3">
<title>Update dmgn status with Network Info</title>

<read_first>
- internal/cli/status.go
- internal/api/handlers.go (status endpoint)
</read_first>

<action>
Update `internal/cli/status.go` to query the local API for live network status instead of showing hardcoded "Not started":

1. Try HTTP GET to `http://localhost:{port}/status` (using cfg.APIPort)
2. If API is reachable, parse the JSON response and display:
   ```
   Network:
     Status: Running
     Peer ID: 12D3KooW...
     Peers: N connected
     Listen: /ip4/0.0.0.0/tcp/XXXX
   ```
3. If API is not reachable, fall back to current display:
   ```
   Network:
     Status: Not started
     Peers: 0 connected
   ```

Also update `internal/api/handlers.go` status endpoint to include network stats when a host reference is available:
- Add `NetworkHost *network.Host` field to `Server` struct (optional, nil when running serve-only mode)
- In status handler, if NetworkHost is not nil, include `"network"` object with peer_id, connected_peers, listen_addrs
</action>
