<objective>
Build a generic knowledge graph / DAG system for DMGN — extract entities from any source (code, memories, text), discover relationships, store as typed edges, and enable queries.

The graph supports any node type and typed edges, unlike embedding similarity which is implicit.
</objective>

<context>
@pkg/memory/memory.go
@pkg/storage/storage.go
@.planning/phases/11-hybrid-link-processing/11-CONTEXT.md

**What we're building:**

A DAG where:
- **Nodes** = entities (programmer, nlg, ai_stuff, concepts, memories, files, functions)
- **Edges** = typed relationships (CREATES, USES, BUILT_BY, PART_OF, RELATED_TO)

Example:
```
programmer --CREATES--> nlg
nlg --BUILT_BY--> programmer
nlg --CREATES--> ai_stuff
ai_stuff --CREATED_BY--> nlg
programmer --USES--> tools
```

**Why this matters:**

Embedding similarity gives implicit connections. Knowledge graph gives explicit, meaningful relationships that can be queried and traversed.
</context>

type Node struct {
    ID        string            `json:"id"`

    Label    string            `json:"label"`    // display name
    Meta     map[string]any   `json:"meta"`    // extra data

```go
type Edge struct {
    ID        string    `json:"id"`
    From      string    `json:"from"`        // source node ID
    To        string    `json:"to"`          // target node ID
    Type      string    `json:"type"`        // "CREATES", "USES", "BUILT_BY", "PART_OF", "RELATED_TO", etc
    Weight    float32   `json:"weight"`
    CreatedAt int64   `json:"created_at"`
}
```
